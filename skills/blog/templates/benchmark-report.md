# Benchmark Report Template（基準報告模板）

> 適用於原創數據分析、基準測試、實驗報告。
> 特點：數據密集、方法論透明、高 E-E-A-T 權重。

## 適用情境

- 自己做的測試/實驗結果
- 產業基準數據分析
- 工具/方案效能比較測試
- 調查問卷結果分析

## 文章結構

### H1: [主題] + [年份] + 基準報告/實測結果
- 範例：「2026 年 AI 寫作工具效能實測：8 款工具的品質與速度比較」

### 引言（150-200 字）
- **Answer-First：** 最重要的發現（一句話）
- 測試規模（N 個樣本/工具/案例）
- 為什麼做這個測試

### TL;DR（60-80 字）
- 核心發現 + 最佳推薦

### H2: 方法論（200-300 字）
- 測試環境/條件
- 評估標準
- 樣本數量
- 時間範圍
- 限制說明
- [原創數據] 標記

### H2: 核心發現 1（300-400 字）
- Answer-First：最重要的數據點
- 詳細數據表格
- [圖表：比較圖]
- 分析解讀
- [原創數據] 標記

### H2: 核心發現 2（300-400 字）
- [同上結構]

### H2: 核心發現 3（300-400 字）
- [同上結構]

### H2: 綜合排名/推薦（300-400 字）
- 總評比較表
- 不同情境的推薦
- [圖表：綜合排名]

### H2: 與去年/歷史數據比較（200-300 字，可選）
- 趨勢分析
- 進步/退步幅度
- [圖表：趨勢線]

### FAQ（3-5 個）
- 圍繞方法論和發現的常見問題
- 每個答案含具體數據

### 結論（150-200 字）
- 核心發現重述
- 實用建議
- 下次測試計畫

## 字數目標

| 語言 | 字數 |
|------|------|
| 英文 | 2,500-3,500 |
| 中文 | 4,000-5,500 |

## 特殊要求

- **原創數據標記：** 每個發現段落都加 `[原創數據]`
- **表格密度：** 至少 2 個數據表格
- **圖表密度：** 至少 3 個圖表
- **方法論透明：** 必須有獨立的方法論段落
- **Schema：** 使用 BlogPosting + Dataset schema
- **E-E-A-T 加分：** 此模板天然符合 Experience + Expertise 維度
