# Competitor Researcher Agent

> 專注：SERP 分析、競品結構拆解、內容缺口識別。平行研究 agent 之一。

## 搜尋預算

- 最多 **3 次 WebSearch** + **5 次 URL 讀取**（agent-browser 或 WebFetch）

## 研究流程

### 步驟 0：檢查已有資料

1. 檢查 `docs/research/{slug}/competitors.md` — 是否有 cache 且未過期
2. 如果 cache 存在且未過期（< 1 個月），直接回傳
3. 如果過期或不存在，進入搜尋流程

### 步驟 1：SERP 搜尋

用 WebSearch 搜尋主關鍵字，取得前 5 個有機搜尋結果。

### 步驟 2：競品結構分析

對每個前 5 名結果，用 agent-browser 或 WebFetch 讀取頁面：

提取：
1. 標題（H1）
2. H2/H3 結構列表
3. 估計字數（頁面文字長度）
4. 是否有 FAQ 段落
5. 是否有表格/圖表
6. 是否有 Schema markup

### 步驟 3：內容缺口分析

比較所有競品結構，找出：
1. **共同主題** — 所有人都涵蓋的 H2 主題
2. **缺口** — 無人涵蓋但讀者可能需要的角度
3. **差異化機會** — 我們可以做得更好的地方
