# E-E-A-T 訊號指南

> 衍生自 STRATEGY.md 1.2 節。作者歸因、信任指標、經驗訊號的實作細節。
>
> **相關文件：** `content-rules.md`（引用格式和來源分級）、`schema-stack.md`（Person Schema）

## E-E-A-T 權重

| 維度 | 建議權重 | 核心要求 |
|------|---------|---------|
| Trust（信任） | 30% | 準確資訊、可查證來源、作者可信度 |
| Experience（經驗） | 25% | 第一人稱、實測結果、原創截圖 |
| Expertise（專業） | 25% | 作者資歷、深度分析、技術準確 |
| Authoritativeness（權威） | 20% | 外部引用、反向連結、品牌知名度 |

> **注意：** Google 從未公布各維度的官方權重。以上為業界共識的建議分配，用於內部評分。

> Trust 永遠最先被評估。如果信任度低，其他三項分數無效。

## 作者歸因（必要）

### 最低要求

每篇文章必須包含：
- **具名作者**（不可用「管理員」「團隊」「Admin」）
- **作者簡介**（50-100 字，含職稱和相關經歷）
- **社群連結**（至少 LinkedIn 或 Twitter）

### Frontmatter 範例

```yaml
author: 許文馨
authorTitle: Pumpfly 數位行銷總監
authorBio: >
  10 年數位行銷經驗，專注 SEO 與內容策略。曾協助超過 50 間企業
  提升自然搜尋流量，對 AI 搜尋優化有深入研究。
authorLinks:
  linkedin: https://linkedin.com/in/example
  twitter: https://twitter.com/example
```

### Person Schema

```json
{
  "@type": "Person",
  "name": "許文馨",
  "jobTitle": "數位行銷總監",
  "worksFor": {
    "@type": "Organization",
    "name": "Pumpfly"
  },
  "sameAs": [
    "https://linkedin.com/in/example",
    "https://twitter.com/example"
  ]
}
```

## 經驗訊號

### 第一人稱表達模式

**好的範例（有具體經驗）：**
> 以下為示範格式，寫作時替換為真實經驗和數據。
- 「我們在 3 個月的測試中發現，[策略名稱] 讓 [指標] 提升了 [N] 倍。」
- 「在我的實際操作中，[工具A] 的 [指標] 比 [工具B] 快 [N] 倍。」
- 「我們團隊分析了 [N] 篇排名前 3 的文章後，歸納出以下模式。」

**不好的範例（空泛）：**
- 「根據業界經驗，SEO 很重要。」
- 「在我們的了解中，這是一個趨勢。」
- 「據我所知，很多人這樣做。」

### 原創性標記

在文章中使用以下標記，強化原創訊號：

- `[原創數據]` — 自行收集的調查、實驗、A/B 測試結果
- `[實際經驗]` — 第一手觀察、操作心得、「我們試了 X，結果是 Y」
- `[獨特見解]` — 別人沒提出的分析角度、反直覺的觀點 + 數據佐證

每篇至少 2 個標記，綜合型文章目標 3 個。

## 信任指標

### 文章內信任元素

| 元素 | 做法 |
|------|------|
| 來源引用 | 每個數據都有 Tier 1-3 來源連結 |
| 方法論揭露 | 說明數據怎麼來的（「我們分析了 200 篇文章」） |
| 更新日期 | frontmatter 中的 `lastUpdated` 欄位 |
| 審核標記 | 「由 [專家名] 審核」（如適用） |
| 利益揭露 | 如有業配或聯盟連結，明確標示 |

### 站點層級信任

- 有「關於我們」頁面，介紹團隊背景
- 有「聯絡我們」頁面
- 有隱私權政策
- HTTPS 加密
- 乾淨的網域歷史

## 依內容類型的 E-E-A-T 重點

| 內容類型 | 最重要的維度 | 建議做法 |
|---------|-------------|---------|
| 教學指南 | Experience | 加入實際操作截圖和步驟成果 |
| 排行榜 | Expertise | 說明評選標準和方法論 |
| 比較文 | Trust | 列出完整比較維度，不偏袒 |
| 案例研究 | Experience | 真實數據、時間軸、具體結果 |
| 長篇指南 | Expertise + Authority | 深度分析 + 引用權威來源 |
