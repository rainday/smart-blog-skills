# 內部連結策略

> 衍生自 STRATEGY.md。連結密度、錨文字、Hub-and-Spoke 模型。
>
> **相關文件：** `content-templates.md`（`pillar-page` 模板 = Hub 頁面）、`content-rules.md`（連結評分標準）

## 連結密度

| 文章長度 | 內部連結數 | 外部連結數 |
|---------|----------|----------|
| 1,500 字以下 | 3-5 | 2-3 |
| 1,500-2,500 字 | 5-8 | 3-5 |
| 2,500-4,000 字 | 8-12 | 5-8 |

---

## 連結放置位置

| 位置 | 連結類型 | 目的 |
|------|---------|------|
| 引言 | 支柱頁面/主題中心 | 建立主題關聯 |
| H2 段落中 | 深度延伸文章 | 提供深入閱讀 |
| FAQ 答案中 | 詳細說明文章 | 引導到完整內容 |
| 結論 | 下一步閱讀 | 引導讀者繼續 |

## 錨文字規則

### 分佈目標

| 錨文字類型 | 比例 | 範例 |
|-----------|------|------|
| 精確匹配 | 5-10% | 「SEO 優化指南」 |
| 部分匹配 | 20-30% | 「這份 SEO 教學」 |
| 語義相關 | 30-40% | 「提升搜尋排名的方法」 |
| 品牌/自然 | 20-30% | 「我們之前的研究」 |

### 禁止的錨文字

- ❌「點此」「點擊這裡」
- ❌「閱讀更多」「了解更多」
- ❌「這篇文章」「這裡」「連結」
- ❌ 純 URL

### 好的錨文字範例

```markdown
想深入了解 AI 搜尋的影響，可以參考我們的
[AI 搜尋優化完整指南](內部連結：AI 搜尋優化 → 支柱頁面)。
```

## Hub-and-Spoke 模型

### 結構

```
        支柱頁面（Hub）
       /    |    \
     /      |      \
  子文章1  子文章2  子文章3  （Spoke）
     \      |      /
      \     |     /
       相互連結
```

### 規則

1. **支柱頁面**（Pillar Page）
   - 廣泛主題的完整指南（3,000-4,000 字）
   - 連結到所有子文章
   - 每個子主題用 1-2 段概述 + 連結到深度文章

2. **子文章**（Spoke）
   - 特定子主題的深度文章
   - 連結回支柱頁面
   - 連結到相關的其他子文章

3. **雙向連結**
   - 支柱 → 子文章 ✅
   - 子文章 → 支柱 ✅
   - 子文章 → 子文章 ✅（相關時）

## 寫作時的標記方式

在文章中用 placeholder 標記內部連結位置：

```markdown
[內部連結：錨文字 → 目標描述]
```

範例：
```markdown
關於標題層級的詳細規則，請參考我們的
[內部連結：SEO 標題優化指南 → H1-H6 標題層級和關鍵字配置的完整教學]。
```

使用者在收到文章後，將 placeholder 替換為實際 URL。

## 常見問題

### 孤兒頁面

沒有任何內部連結指向的頁面。解決方式：
- 定期掃描所有文章的連結
- 新文章發布時，回頭在舊文章加入連結
- 確保每篇文章至少被 2 篇其他文章連結

### 主題蠶食（Cannibalization）

多篇文章針對同一個關鍵字競爭。解決方式：
- 選一篇作為主要文章，其他文章連結到它
- 或合併成一篇更完整的文章
- 檢查方式：在站內搜尋關鍵字，看哪些文章出現
