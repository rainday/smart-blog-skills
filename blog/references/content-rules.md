## 寫作規則

### Answer-First 格式（最重要的單一規則）

每個 H2 段落的前 40-60 字（中文 60-100 字）必須：
- 包含一個具體統計數字 + 來源歸因
- 直接回答標題隱含的問題

範例：
```markdown
## AI 搜尋如何影響自然流量？

根據 Seer Interactive 的 2025 年研究，AI Overview 出現時自然搜尋
點擊率下降了 61%（[Seer Interactive](https://seerinteractive.com), 2025）。
這代表內容必須同時為搜尋引擎和 AI 系統優化。
```

### 段落規則

| 規則 | 中文 | 英文 |
|------|------|------|
| 段落長度 | 60-120 字 | 40-80 字 |
| 絕對上限 | 200 字 | 150 字 |
| 句子長度 | 15-25 字 | 15-20 字 |
| 句子上限 | 35 字 | 25 字 |

- 每段以最重要的資訊開頭
- Flesch Reading Ease 目標：60-70（英文）

### 標題規則

- **H1**：僅文章標題，含主關鍵字，40-60 字元
- **H2**：主要段落標題，60-70% 用問句格式
- **H3**：子段落，僅在 H2 下使用
- **禁止跳級**：H2→H4 是錯的，必須 H2→H3→H4
- 主關鍵字自然出現在 2-3 個標題中

### TL;DR 區塊

放在引言之後、第一個 H2 之前：
```markdown
> **TL;DR：** [40-60 字自成一體的摘要，含 1 個統計數字 + 來源。
> 讀者只看這段就能獲得核心價值。]
```

### 引用格式

行內引用（統一格式）：
```markdown
[統計數字]（[來源名稱](URL), YYYY）
```

### 內部連結

- 每 2,000 字 5-10 個內部連結
- 使用描述性錨文字（非「點此」「閱讀更多」）
- 在引言、H2 段落、FAQ、結論中放置連結
- 用 placeholder 標記：`[內部連結：錨文字 → 目標描述]`

### 圖片規範

- 每 200-400 字放一張圖片
- 封面圖 1200x630（OG 相容）
- Alt 文字用完整描述句，自然包含關鍵字
- Hero 圖不可 lazy load

## 反 AI 偵測

### 必做

- 句長「爆發性」變化（短句 8 字 + 長句 25 字交替）
- 口語化縮寫和語助詞
- 每 200-300 字一個修辭問句
- 第一人稱經驗（「我們測試後發現...」）

### 禁止觸發詞

**中文：**
在當今數位時代、值得注意的是、深入探討、全面性指南、
無縫整合、顛覆性、賦能、一站式、不可或缺、至關重要

**英文：**
delve, tapestry, multifaceted, testament, pivotal, robust,
cutting-edge, leverage, comprehensive, landscape, crucial,
foster, illuminate, paramount, nuanced, intricate, meticulous, seamlessly

英文觸發短語：
in today's digital landscape, it's important to note,
game-changer, navigate the landscape

### 偵測指標

| 指標 | 自然 | 警告 | AI 風險 |
|------|------|------|--------|
| Burstiness（句長變異） | ≥0.4 | 0.3-0.4 | <0.3 |
| TTR（詞彙多樣性） | ≥0.4 | 0.35-0.4 | <0.35 |
| 觸發詞/千字（中文） | ≤3 | 3-5 | >5 |
| 觸發詞/千字（英文） | ≤5 | 5-8 | >8 |
| 被動語態 | ≤10% | 10-15% | >15% |

## 100 分品質評分

### 內容品質（30 分）

| 項目 | 滿分 | 評分標準 |
|------|------|---------|
| 深度 | 7 | 字數在內容類型基準範圍內 |
| 可讀性 | 7 | Flesch 60-70 = 7分，55-75 = 5分 |
| 原創性 | 5 | 有原創數據/經驗標記 = 5分 |
| 結構 | 4 | H2≥3 + 層級正確 + 段落分佈均勻 |
| 參與度 | 4 | 正文含問句≥2 + 範例≥2 |
| 文法/反模式 | 3 | 句長變異 + 無超長句 + 轉折詞適當 |

### SEO 優化（25 分）

| 項目 | 滿分 | 評分標準 |
|------|------|---------|
| 標題 | 4 | 40-60 字元含關鍵字 |
| 標題層級 | 5 | H1=1 + H2≥3 + 不跳級 + 有 H3 |
| 關鍵字佈局 | 4 | 標題+首段+標題中出現 |
| 內部連結 | 4 | 依字數：1,500 字以下 3-5 個；1,500-2,500 字 5-8 個；2,500+ 字 8-12 個 |
| Meta 描述 | 3 | 150-160 字元含數據 |
| 外部連結 | 2 | ≥2 個 Tier 1-2 來源 |
| URL 結構 | 3 | 短、含連字號、無長數字串 |

### E-E-A-T 訊號（15 分）

| 項目 | 滿分 | 評分標準 |
|------|------|---------|
| 作者歸因 | 4 | 具名作者（非通用名） |
| 來源引用 | 4 | ≥5 個引用 + Tier 1 來源 |
| 信任指標 | 4 | 有「關於」連結 + 編輯/審核標記 |
| 經驗訊號 | 3 | ≥3 處第一人稱經驗描述 |

### 技術元素（15 分）

| 項目 | 滿分 | 評分標準 |
|------|------|---------|
| Schema | 4 | BlogPosting + Person + FAQPage |
| 圖片 | 3 | 全部有 alt + 現代格式 |
| 結構化資料 | 2 | 有表格 + 清單 |
| 頁面速度 | 2 | lazy load + 現代圖片格式 |
| 行動裝置 | 2 | 段落適當 + 響應式元素 |
| 社群 Meta | 2 | OG tags + 社群圖片 |

### AI 引用就緒度（15 分）

| 項目 | 滿分 | 評分標準 |
|------|------|---------|
| 可引用段落 | 4 | ≥5 個 120-180 字自成一體段落 |
| Q&A 格式 | 3 | ≥5 個問答對或有 FAQ |
| 實體清晰度 | 3 | ≥3 個 **粗體術語** + 定義 |
| 提取結構 | 3 | 有 TL;DR + 表格 + 清單 |
| 爬蟲可及性 | 2 | 無 robots 限制 |

### 評分等級

| 分數 | 等級 | 行動 |
|------|------|------|
| 90-100 | 卓越 | 直接發布 |
| 80-89 | 優良 | 微調即可 |
| 70-79 | 及格 | 針對性改善 |
| 60-69 | 待改進 | 大幅修改 |
| <60 | 重寫 | 從大綱重新開始 |

### 品質閘門（發布前必通過）

| 規則 | 等級 |
|------|------|
| 無捏造統計 | 🔴 致命 |
| 標題層級正確 | 🔴 致命 |
| 具名作者 | 🔴 致命 |
| Answer-First 格式 | 🟡 高 |
| TL;DR 區塊 | 🟡 高 |
| Meta Description | 🟡 高 |
| ≥8 個有來源統計 | 🟡 高 |
| 3-5 個 FAQ | 🟠 中 |
| 3-5 張圖片 | 🟠 中 |
| 內部連結（依字數） | 🟠 中 |
